데이터 처리 란?
- 데이터를 기존의 형식으로 부터 다음 형식으로 변환하는 과정
    -> 이것을 데이터 변환이라고 한다면 데이터 처리란 정보를 데이터로 혹은 데이터를 정보로 변환하는 과정
================================================================

예외 처리 란?
- 실행 흐륾상 오류가 발생 하였을 때, 오류를 그대로 내보내지 않고 오류에 대응하는 방법을 제시하는 개념이나 하드웨어 구조를 뜻한다.
===============================================================

자주 접하는 오류 종류
    
    ReferenceError ,예외처리

    - Uncaught ReferenceError : oo is not defined
        oo부분을 잘못 입력했을 때 발생
    
    SyntaxError , 구문오류

    - Uncaught SyntaxError : invalid or unexpected token
        토큰(기호)을 잘못 입력했을 때의 오류가
    
    - Uncaught SyntaxError : missing ) after argument list
        괄호를 제대로 닫지 않음

    - Uncaught SyntaxError : Unexpected identifier
        식별자가 예상하지 못한 위치에서 등장했다는 오류

    
=====================================================

표현식(expression) 이란?
    - 값을 만들어 내는 간단한 코드

문장(statement) 이란?
    - 하나 이상의 표현식이 모인것

프로그램(program) 이란?
    - 여러 문장이 모여 만들어짐

키워드(keyword) 란?
    - 정해져있는 특별한 의미가 있는 단어

식별자(identifier) 란?
    - 프로그래밍 언어에서 이름을 붙일 때 사용하는 단어
        -> 클래스의 이름은 대문자로 시작
        -> 변수와 인스턴스, 함수, 메소드의 이름은 소문자로 시작
        -> 여러 단어로 이루어진 식별자는 각 단어의 첫글자를 대문자로

식별자의 종류
    - 구분                      단독으로 사용       다른 식별자와 사용
    식별자 뒤에 괄로 없음           변수                속성
    식별자 뒤에 괄호 있음           함수                메소드

자료(data) 란
    - 프로그래밍에서 프로그램이 처리할 수 있는 모든것 (자료 형태에 따라 나눠 놓은 것을 자료형(data type0)이라고 한다)
    -> 자료형 종류
        문자열 자료형
        숫자 자료형
        불(boolean) 자료형 ( 참과 거짓 값을 표현할 때)
            - 비교 연산자
                ( === 양쪽이 같다.)
                ( !== 양쪽이 다르다.)
                (> 왼쪽이 더 크다 / < 오른쪽이 더 크다 / >= 왼쪽이 더 크거나 같다 / <=오른쪽이 더 크거나 같다)
            - 불 부정 연산자
                > !true
                false
            
            - 불 논리합/논리곱 연산자
                && 논리곱 연산자 : 양쪽 변의 값이 모두 true 일 때 true를 결과로 냄
                || 논리합 연산자 : 양쪽 변의 값 중 하나만 true 여도 true를 결과로 냄

*
    ===와 !== 연산자 => '값'과 '자료형'이 같은지 를 비교하는 연산자
    ==와 != 연산자는 '값'이 같은지 비교하는 연산자
